Astro Open Infrastructure

An open framework for expressing, verifying, routing and proving blockchain intents without surrendering private keys.

Current status: an unincorporated international open-protocol initiative. It is not a registered company, foundation, charity or government-recognized standards body.

Founding Charter · Version 0.1

Infrastructure in the public interest

Human authorization remains above autonomous execution. Public specifications define how independent products and operators interoperate.

Human control

Every asset-affecting action remains bound to explicit authorization.

Non-custodial

Private keys remain under the control of users and compatible wallets.

Privacy by proof

Public fingerprints minimize exposed identity and transaction material.

Open specifications

Protocols, compatibility tests and security disclosures are public.

Interoperability

Implementations remain portable and avoid forced token dependency.

Verifiable claims

Running capabilities, test activity and future plans remain clearly separated.

Ecosystem framework

Five systems, clear responsibilities

AstroBridge

Safety assessment, intent routing and execution coordination.

AstroChat

Human-to-human and human-to-agent intent entry.

AstroWallet AI

Non-custodial asset control and signature entry.

AstroAI Chain

Intent fingerprints, contribution records and execution proofs.

AstroRealms

Mobile node participation and world-state experience.

Governance

Open decisions with recorded rationale

Initial bodies are defined by responsibility. Membership and voting records must be published before any body claims active representative authority.

Protocol Council

Maintains RFCs, compatibility rules, versioning and public technical rationale.

Security Council

Coordinates vulnerability intake, emergency response and independent audits.

Node Operator Assembly

Represents mobile, home and edge-server operators in network policy.

Contributor Community

Connects developers, designers, translators, researchers and educators.

  1. Publish a proposal as an Astro RFC.
  2. Announce a minimum public review period.
  3. Document security and compatibility impact.
  4. Record votes, rationale and dissenting opinions.
  5. Assign accepted changes a protocol version and test vectors.
Open specifications

Astro RFC index

RFC-0001

Astro Intent Proof Protocol

The non-custodial path from signed intent to safety assessment, edge contribution and verifiable execution proof.

Draft · v0.1

RFC-0001 describes the current reference architecture. It has not received an independent production security audit and is not presented as a finalized international standard.

Contribute

Build the protocol in public

Participation is open across protocol engineering, security, nodes, SDKs, research, translation and technical education.

  • Submit RFCs and compatibility tests.
  • Operate test nodes and publish reproducible reliability data.
  • Add wallet, agent, game and chain integrations.
  • Translate specifications and improve technical education.
  • Report vulnerabilities privately before disclosure.

Never submit private keys, production secrets or personal wallet records. Contribution accounting is not a promise of market value or investment return.
