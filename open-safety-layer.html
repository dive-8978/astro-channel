Shared safety boundary

One check between intent and execution

Wallets, dapps, agents and bridges can use the same machine-readable decision without handing Astro control of user assets.

01 · ENVELOPE

Normalize actor, actions, constraints, nonce and deadline.

02 · FINGERPRINT

Generate a deterministic SHA-256 identity for the complete intent.

03 · SIMULATE

Run static policy checks and optional allowlisted RPC calls.

04 · ASSESS

Return explainable signals and allow, review or block.

05 · RECEIPT

Sign the result using a published Ed25519 verification key.

Deterministic policy

Safety decisions with evidence

The reference engine checks what it can prove and labels what it cannot. Unknown never silently becomes safe.

  • DestinationsInvalid EVM and zero-address detection.
  • PermissionsERC-20 unlimited approval and operator-wide NFT approval detection.
  • ConstraintsDeadline, aggregate value and human approval thresholds.
  • SimulationOptional eth_call and gas estimation through configured RPC endpoints.
  • Threat feedEvidence submissions remain pending until reviewed.

Open API · v0.1

Small surface, portable integration

The TypeScript reference implementation includes OpenAPI discovery, an SDK, Docker deployment and request controls.

MethodEndpointPurpose
POST/v1/intents/fingerprintValidate and fingerprint an AOI intent envelope.
POST/v1/simulateRun deterministic checks and optional chain simulation.
POST/v1/checkReturn simulation plus an explainable safety decision.
POST/v1/routeRank integrator-supplied routes by fee, speed and safety.
POST/v1/receiptsIssue an Ed25519-signed safety receipt.
POST/v1/receipts/verifyVerify receipt integrity against the published key.
GET/v1/safety/{chain}/{subject}Query the open safety feed without assuming unknown is safe.
